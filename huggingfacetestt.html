How It Works

This app uses the Hugging Face Inference API to perform AI sentiment analysis. Here's exactly how the code works — so you can build something similar yourself.

The API Call

It's just a fetch() request. One function, fully reusable in any web or mobile app.

Copy the code
The steps
1

Get a free Hugging Face token

Create a free account at huggingface.co, go to Settings → Tokens, and create a token with Read access. This is your key to the API — keep it private, never put it in public code.

2

Choose a model

Hugging Face hosts thousands of open-source AI models. This app uses distilbert-base-uncased-finetuned-sst-2-english — a lightweight model trained specifically for sentiment analysis. The model URL becomes part of your fetch endpoint.

3

Make a POST request

Send your text as JSON in the request body. Include your token in the Authorization header as a Bearer token. The API processes your text through the model and returns a JSON response.

4

Read the JSON response

The API returns an array of objects, each with a label (POSITIVE or NEGATIVE) and a score (0 to 1). Your code reads this data and updates the UI — that's the whole app.

The actual fetch code
// 1. Define the model endpoint
const MODEL_URL = "https://api-inference.huggingface.co/models/distilbert-base-uncased-finetuned-sst-2-english";

// 2. Make the API call
async function analyseSentiment(text, token) {

  const response = await fetch(MODEL_URL, {
    method: "POST",
    headers: {
      "Authorization": `Bearer ${token}`,   // Your HF token
      "Content-Type": "application/json"
    },
    body: JSON.stringify({ inputs: text })  // The text to analyse
  });

  const data = await response.json();

  // 3. data looks like this:
  // [[{ label: "POSITIVE", score: 0.9998 },
  //   { label: "NEGATIVE", score: 0.0002 }]]

  return data[0];  // Array of { label, score } objects
}
Why this matters

This exact fetch() pattern works identically in:
Plain HTML/JS · React · React Native (mobile app) · Node.js (server)

Change the MODEL_URL to use a completely different AI capability — text generation, image classification, translation — with no other changes to your code structure. The pattern is always the same.
